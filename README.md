# Shard

`Shard`는 중앙 결정체를 거점으로 삼아 파편을 채굴하고, 적의 공세를 막아내며, 성장 선택을 누적해 보스 사이클을 돌파하는 거점 방어형 액션 게임 프로젝트입니다.

이 프로젝트에서는 단순히 전투 요소를 추가하는 데 그치지 않고, `채굴`, `성장`, `방어`, `복귀`, `보스전`, `스테이지 전환`이 하나의 게임플레이 루프로 맞물리도록 설계하고 구현하는 데 집중했습니다.

## 한 줄 소개

파편 채굴과 적 방어를 분리된 콘텐츠로 두지 않고, 서로의 보상과 리스크가 다음 선택으로 이어지도록 구성한 거점 방어형 액션 게임입니다.

## 핵심 게임플레이 루프

1. 결정체 주변의 파편 블록을 파괴해 자원을 획득한다.
2. 적을 처치해 추가 전투 재화를 확보한다.
3. 결정체로 복귀해 코어, 플레이어, 동료 전력을 강화한다.
4. 파편 파괴 진행도가 누적되면 보스가 등장한다.
5. 보스를 처치하면 전장을 복원하고 다음 스테이지로 넘어간다.
6. 마지막에는 결정체 과부하를 버텨내는 엔딩 페이즈를 수행한다.

이 루프에서 중요한 점은, 각 행동이 독립적으로 존재하지 않는다는 점입니다.

- 파편 채굴은 성장 재화 수급일 뿐 아니라 보스전 도달 속도를 당깁니다.
- 적 처치는 위협 제거에 그치지 않고 동료 전력 확장으로 이어집니다.
- 결정체 복귀는 단순 상점 이용이 아니라 다음 전투 양상을 결정하는 선택 구간입니다.

`Shard`의 핵심 목표는 플레이어가 계속 판단하도록 만드는 것입니다.

- 지금 더 바깥으로 나가 파편을 채굴할 것인가
- 결정체로 돌아가 현재 재화를 성장에 투자할 것인가
- 적을 더 정리해 영혼을 모을 것인가
- 보스 등장 전에 빌드를 더 완성할 것인가

즉, 이 프로젝트는 액션의 손맛만이 아니라 `판단이 누적되는 전투 루프`를 만드는 데 초점을 둔 프로젝트입니다.

## 핵심 시스템

<br />

### 1. 이중 재화 기반 성장 구조

성장 시스템은 하나의 자원으로 모든 선택을 해결하지 않도록 구성했습니다.

- 파편/에너지: 블록 파괴로 획득, 코어 및 플레이어 강화에 사용
- 영혼: 적 처치로 획득, 동료 전투기 소환 및 강화에 사용
- 경험치: 전투와 채굴을 통해 획득, 레벨업과 능력 선택에 사용

이 구조를 통해 플레이어는 한 행동만 반복해서는 최적 성장을 만들 수 없습니다.  
바깥으로 나가 채굴해야 하고, 다시 돌아와 투자해야 하며, 방어를 통해 전투 보상도 챙겨야 합니다.

결과적으로 성장 자체가 `공간 이동`, `전투 리스크`, `복귀 타이밍`과 결합되도록 설계했습니다.

</details>

<summary><strong>2. 이벤트 기반 능력 시스템</strong></summary>

### 2. 이벤트 기반 능력 시스템

능력 시스템은 전투 이벤트에 반응하는 형태로 구성했습니다.

- 능력은 `Attack`, `Critical`, `Kill`, `Passive` 같은 이벤트 타입에 연결됩니다.
- 전투 결과는 `AttackData`를 통해 전달됩니다.
- 각 능력은 이벤트에 반응해 추가 공격, 상태 이상, 처치 보상, 조건부 효과를 실행합니다.

이 방식의 목적은 명확합니다.

- 능력을 단순 수치 증가로 소비하지 않기
- 플레이어의 전투 양상에 따라 체감이 달라지도록 만들기
- 이후 새로운 능력을 추가하더라도 기존 전투 구조를 크게 깨지 않고 확장 가능하게 유지하기

즉, 능력은 장비 목록이 아니라 `전투 규칙을 바꾸는 파츠`로 작동하도록 의도했습니다.

</details>

<summary><strong>3. 보스와 스테이지 전환이 연결된 진행 구조</strong></summary>

### 3. 보스와 스테이지 전환이 연결된 진행 구조

보스전은 별도의 이벤트가 아니라 전체 루프의 일부로 동작하도록 구성했습니다.

- 기본 웨이브는 외곽 스폰 포인트에서 반복 생성됩니다.
- 파괴된 블록 수가 누적되면 보스 등장 조건이 충족됩니다.
- 보스 처치 후에는 결정체 복귀, 전장 복원, 다음 스테이지 전환이 이어집니다.
- 최종 스테이지에서는 과부하 타이머와 대량 스폰을 결합해 엔딩 페이즈를 구성했습니다.

이 구조 덕분에 보스전은 갑자기 끊겨 들어오는 콘텐츠가 아니라, 플레이어가 이전까지 축적한 선택의 결과로 도달하는 중간 결산 역할을 하게 됩니다.

</details>

<details>
<summary><strong>4. 전장 복원 구조</strong></summary>

<br />

### 4. 전장 복원 구조

이 프로젝트의 중요한 정체성 중 하나는 전장을 일회성 공간으로 소비하지 않는다는 점입니다.

- 플레이어는 파편을 파괴하며 전장을 바꿉니다.
- 보스 처치 후 결정체가 전장을 복원합니다.
- 복원된 공간은 다음 스테이지의 새로운 자원 상태와 압박 구조로 다시 사용됩니다.

이 설계를 통해 같은 공간을 반복 사용하더라도, 단순 재탕이 아니라 `리셋된 다음 국면`처럼 느껴지도록 만들고자 했습니다.

</details>

## 구현 관점에서 중요했던 점

<details>
<summary><strong>구현 관점에서 중요했던 점 자세히 보기</strong></summary>

<br />

이 프로젝트에서 코드 구현보다 먼저 중요했던 것은, 각 시스템이 어떤 플레이 감각을 만들어야 하는지 정의하는 일이었습니다.

특히 아래 기준을 명확히 두고 작업했습니다.

- 채굴은 수집 행위가 아니라 전투 페이스를 조절하는 행동이어야 한다.
- 적 처치는 생존 수단이면서 다음 전력 확장으로 이어져야 한다.
- 성장은 메뉴 선택이 아니라 전장 운영의 결과로 얻어져야 한다.
- 보스전은 단절된 이벤트가 아니라 루프의 압축된 결과여야 한다.
- 엔딩은 단순 클리어 컷신이 아니라, 세션 전체의 서사적 보상처럼 작동해야 한다.

README를 보는 입장에서 중요하게 봐야 할 포인트도 여기에 있습니다.  
이 프로젝트는 기능 수를 늘리는 방향보다, `각 기능이 어떤 플레이 경험을 만들기 위해 존재하는지`를 연결하는 방향으로 구현되었습니다.

</details>

## 사용 기술

- Unity 6 (`6000.3.11f1`)
- C#
- Universal Render Pipeline
- Unity Input System
- Unity AI Navigation
- UGUI / TextMesh Pro
- Object Pooling

## 관련 구현 예시

- `AbilityManager`: 이벤트 기반 능력 등록 및 발동
- `PlayerLevelManager`: 경험치 누적, 레벨업, 능력 선택 트리거
- `Inventory`: 파편과 영혼 재화 분리 관리
- `CoreInteractUI`: 재화 소비와 성장 선택 연결
- `SpawnManager`: 웨이브 진행, 보스 등장, 엔딩 스폰 처리
- `StageManager`: 보스 처치 이후 스테이지 복원 및 엔딩 페이즈 전환
- `PoolingManager`: 반복 생성 오브젝트 재사용

## 회고

`Shard`를 통해 가장 집중해서 다룬 것은 "재미를 만드는 규칙을 어떻게 시스템으로 고정할 것인가"였습니다.

액션 게임은 타격감만으로 설명되지 않습니다.  
플레이어가 무엇을 위해 싸우는지, 왜 이동하는지, 언제 복귀하는지, 어떤 위험을 감수할지 스스로 판단하게 만들어야 루프가 살아납니다.

이 프로젝트에서는 그 판단 구조를 아래처럼 묶으려 했습니다.

- 채굴은 성장으로 이어진다.
- 방어는 동료 전력 확장으로 이어진다.
- 복귀는 다음 국면의 선택으로 이어진다.
- 보스전은 지금까지의 선택을 검증한다.
- 엔딩은 세션 전체를 보상한다.

결과적으로 `Shard`는 전투 시스템 몇 개를 구현한 프로젝트라기보다,  
`플레이어의 선택이 계속 다음 플레이를 바꾸는 게임플레이 루프를 설계하고 코드로 구현한 프로젝트`로 정리할 수 있습니다.

</details>
