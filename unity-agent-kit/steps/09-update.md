# Step 09 — 하네스 업데이트

사용자가 "하네스 업데이트해줘" + 킷 경로를 알려주면 이 절차를 따른다.

---

## 1단계 — 버전 확인

프로젝트 루트의 `harness-version` 파일에서 `kit-commit` 값을 확인한다.

```
kit-commit: abc1234
installed: 2026-03-01
kit-path: C:\...\unity-agent-kit
```

파일이 없으면: 구버전 설치로 간주. 전체 비교 모드로 진행.

---

## 2단계 — 변경 파일 목록

```bash
# 변경된 파일 목록
git -C [킷경로] diff --name-only [kit-commit]..HEAD

# 커밋 요약
git -C [킷경로] log --oneline [kit-commit]..HEAD
```

파일이 없었으면: `git diff --name-only`로 현재 킷 전체 템플릿 나열.

---

## 3단계 — 분류 및 계획

| 분류 | 대상 파일 | 처리 |
|------|-----------|------|
| **자동 적용** | 하네스 문서, 도구 스크립트, MCP 가이드 (프로젝트 커스터마이징 없는 것) | 직접 덮어쓰기 |
| **병합 필요** | AGENTS.md, CLAUDE.md, AgentPromptTemplates.md 등 프로젝트 맞춤 내용 포함 | 새 섹션만 추출해 기존 내용 보존하며 병합 |
| **검토 필요** | 경로 플레이스홀더 포함, 프로젝트 구조 의존 파일 | 사용자에게 변경 내용 보고 후 판단 |

계획을 사용자에게 제시하고 승인받은 뒤 실행한다.

---

## 4단계 — 적용

- 자동 적용: 복사
- 병합 필요: 기존 파일 읽기 → 새 섹션 식별 → 프로젝트 맞춤 내용 보존하며 갱신
- 검토 필요: 사용자 확인 후 처리

---

## 5단계 — 버전 파일 갱신

```
kit-commit: [새 커밋 해시]
installed: [오늘 날짜 YYYY-MM-DD]
kit-path: [킷 경로]
agent: claude-code | codex
```

---

```
- [항목]
```
