# ProjectWiki

이 디렉터리는 에이전트가 프로젝트를 작업하면서 발견한 지식을 누적하는 위키다.
에이전트가 작성하고 유지한다. 사람은 읽고 질문한다.

`docs/AgentHandoffs/**`와 역할이 다르다.

- `ProjectWiki`는 오래 남길 지식
- `AgentHandoffs`는 다음 에이전트에게 바로 넘길 짧은 작업 인계

## 구조

```
docs/ProjectWiki/
├── README.md          ← 이 파일. 위키 운영 규칙
├── index.md           ← 전체 페이지 카탈로그 (항법용)
├── log.md             ← 작업 이력 (시간순 append-only)
├── systems/           ← 런타임 시스템 페이지
├── assets/            ← 씬/프리팹/ScriptableObject 페이지
└── adr/               ← 아키텍처 결정 기록
```

## 언제 페이지를 만드는가

| 상황 | 만들 페이지 |
|------|------------|
| 시스템을 조사하거나 수정했을 때 | `systems/<SystemName>.md` |
| 씬/프리팹/에셋 의존성을 파악했을 때 | `assets/<AssetName>.md` |
| 아키텍처 결정 이유를 추론/확인했을 때 | `adr/ADR-<NNN>-<slug>.md` |
| 조사 결과가 재사용 가치가 있을 때 | 적절한 카테고리에 자유 형식으로 |

## 페이지 작성 규칙

- 각 페이지 상단에 `_template.md`의 frontmatter를 유지한다.
- 새 페이지를 만들면 반드시 `index.md`에 한 줄 추가한다.
- 작업이 끝나면 `log.md`에 항목을 append한다.
- 기존 페이지와 모순되는 내용을 발견하면 두 페이지 모두 갱신하고 log에 기록한다.
- 페이지는 짧게 유지한다. 길어지면 분리한다.
- Obsidian을 쓴다면 문서 간 연결이 실제로 유용할 때만 `[[wiki link]]`를 추가한다.
- note 이름은 자주 바꾸지 않는다. 링크 안정성이 우선이다.
- handoff 목적으로 `log.md`를 inbox처럼 쓰지 않는다.

## Lint (주기적 건강 점검)

작업 도중 또는 사용자 요청 시 아래를 점검한다.

- index.md에 없는 페이지가 있는지 확인한다.
- log.md에서 오래된 항목 중 현재 코드와 어긋난 내용이 있는지 확인한다.
- 다른 페이지에서 참조되지 않는 고립 페이지를 찾는다.
- AGENTS.md, CLAUDE.md의 폴더 경로/클래스명이 실제 프로젝트와 일치하는지 확인한다.
- `index.md`가 허브 note 역할을 계속 하고 있는지 확인한다.

## 이 위키의 한계

- 에이전트가 직접 확인한 것만 신뢰한다. 추론은 명시적으로 표시한다.
- 코드가 바뀌면 위키도 낡는다. Lint로 주기적으로 갱신한다.
- 원본 소스(코드, 씬, 프리팹)와 모순되면 원본이 우선이다.
