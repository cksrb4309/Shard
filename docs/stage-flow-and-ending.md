# Shard 스테이지 흐름 및 엔딩 구조

## 문서 목적
- 한 세션이 어떻게 시작되고, 반복되고, 끝나는지 구조적으로 정리한다.
- 플레이 리듬, 전환 조건, 엔딩 연출의 기준 문서로 사용한다.

## 연관 문서

- [적 웨이브 및 보스 계획](./enemy-wave-and-boss-plan.md)

## 세션 개요
`Shard`의 세션은 크게 네 구간으로 나뉜다.

1. 초반 성장 및 파편 파괴
2. 웨이브 방어와 보스 등장
3. 결정체 복귀 후 전장 복원과 다음 레벨 전환
4. 세 번째 보스 이후 최종 방어와 정화 엔딩

## 기본 스테이지 흐름

### 1. 시작 구간
- 플레이어는 중앙 결정체를 기준으로 전장을 출발한다.
- 현재 구현에서는 몬스터 스폰이 시작되고, 짧은 초기 지연 뒤 첫 웨이브가 접근한다.
- 이 구간의 목적은 파편 파괴를 시작하고, 첫 성장 선택지를 확보하는 것이다.

### 2. 파편 채굴 및 중간 전투
- 플레이어는 결정체 주변 파편을 부수며 자원을 모은다.
- 동시에 적은 결정체 방향으로 압박을 가한다.
- 파편 파괴량은 단순 재화 획득이 아니라 보스 등장 조건에도 연결된다.

### 3. 보스 등장
- 현재 구현에서는 남은 블록 수가 특정 기준보다 낮아지면 보스가 등장한다.
- 보스는 전용 스폰 포인트와 경고 연출을 사용한다.
- 이 구간은 해당 레벨의 성장 체크포인트다.

### 4. 보스 처치 후 복귀
- 보스를 처치했다고 바로 다음 레벨로 넘어가지는 않는다.
- 플레이어는 반드시 결정체로 돌아와야 한다.
- 이 설계는 "전장을 밀어붙이는 흐름"과 "거점에 다시 연결되는 흐름"을 분명하게 나눈다.

### 5. 전장 복원 및 다음 레벨 전환
- 플레이어가 조건을 만족하면 결정체가 파편 지대를 복원한다.
- 현재 구현에는 색 변화, 파티클, 복원 애니메이션, 추가 난이도 상승이 포함된다.
- 같은 전장을 재사용하면서도 새로운 페이즈로 넘어간다는 감각을 만든다.

## 현재 구현 기준 세부 메모

### 보스 전환 조건
- `SpawnManager`는 전체 블록 수를 추적한다.
- 파괴된 블록이 누적되어 남은 블록 수가 기준 이하가 되면 보스 스폰으로 넘어간다.

### 복귀 판정
- `SearchNearPlayers`는 결정체 주변 원형 영역 충전 방식으로 작동한다.
- 구조상 여러 플레이어가 모두 범위 안에 들어와야 다음 레벨이 열리는 확장이 가능하다.
- 현재 프로젝트가 1인 플레이 중심이더라도, 이 설계는 향후 협동 구조를 붙이기 좋은 형태다.

### 전장 리셋
- `BlockGroup.StageClear()`가 복원 연출과 함께 블록을 다시 활성화한다.
- 현재 구현에서는 다음 스테이지에서 블록 체력과 보상도 상승한다.
- 즉, 전장은 초기화되지만 의미 없이 되감기지는 않고 한 단계 강화되어 돌아온다.

## 엔딩 구조

### 진입 조건
- 1레벨부터 3레벨까지 모두 클리어한다.
- 즉, 보스를 총 3회 처치한 뒤 최종 페이즈에 진입한다.

### 엔딩 페이즈 흐름
1. 결정체에 충분한 에너지가 모였다는 알림이 출력된다.
2. 결정체가 과부하 또는 정화를 준비한다는 메시지가 이어진다.
3. 적들이 이상을 감지하고 결정체로 몰려온다.
4. 플레이어는 마지막 방어 구간을 버틴다.
5. 시간이 지나면 결정체가 폭발적으로 에너지를 방출한다.
6. 주변 적과 파편이 제거되며 엔딩 연출로 이어진다.

### 현재 구현 기준 수치 메모
- 엔딩 전용 스폰이 별도로 존재한다.
- 엔딩 웨이브는 시간이 지날수록 스폰 주기가 더 짧아진다.
- 현재 구현상 결정체 폭발은 방어 시간을 버티면 발동하는 구조다.

## 리듬 분석

### 왜 이 흐름이 좋은가
1. 파편 채굴과 방어가 교차되며 템포가 단조롭지 않다.
2. 보스 처치 후 즉시 종료가 아니라 결정체 복귀를 요구해 거점의 의미가 살아난다.
3. 파편 복원이 있어도 완전 초기화가 아니라 단계 상승이므로 누적 성장감이 유지된다.
4. 최종 엔딩이 "쌓아 온 에너지의 해방"으로 연결되어 세션 전체의 서사 구조가 명확하다.

### 주의할 점
1. 복귀 구간이 너무 길면 전투가 끊긴 느낌이 날 수 있다.
2. 보스 처치 후 다음 페이즈로 넘어가는 조건이 불명확하면 플레이어가 헤맬 수 있다.
3. 엔딩 조건이 시간 버티기인지, 적 처치인지, 충전 게이지인지 분명하지 않으면 결말의 인상이 약해진다.

## 권장 설계 방향

### 일반 스테이지
- 매 스테이지마다 파편 지형의 느낌이나 압박 방향이 조금씩 달라지는 연출을 주는 편이 좋다.
- 복귀 구간에는 짧은 경고 연출이나 잔존 적 처리 구간을 넣어 긴장감을 유지할 수 있다.

### 엔딩
- 결정체 충전 상태를 시각적으로 누적 표시하면 마지막 폭발의 기대감이 커진다.
- 최종 방어에서는 플레이어가 "이제 끝이 보인다"는 정보를 충분히 받아야 한다.
- 정화 폭발 직전 5초 구간은 사운드, 화면, UI 모두 최대치로 밀어주는 편이 좋다.

## 정리 필요 항목
1. 복귀 시 결정체 상호작용이 자동인지, 버튼 입력인지, 일정 시간 유지인지 확정해야 한다.
2. 엔딩 성공 조건을 시간 기반으로 유지할지, 적 처치 기반으로 바꿀지 결정해야 한다.
3. 각 스테이지의 차별점을 색 변화 외에 어떤 규칙 변화로 줄지 정해야 한다.
4. 실패 시 어디서 재시작하는지, 세션 전체를 다시 해야 하는지 구조를 정의할 필요가 있다.
