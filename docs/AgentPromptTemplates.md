# Shard 에이전트 프롬프트 템플릿

## 이 파일의 목적

이 프로젝트에서 프롬프트 단독으로 하네스가 작동하지 않는다.

하네스는 아래 조각들이 합쳐질 때 작동한다:

- 작업 요청
- `AGENTS.md`
- 자동화된 검증
- 작업별 테스트 또는 완료 조건 확인

이 파일은 Codex와 Claude Code 모두 참조할 수 있다.

매번 전체 템플릿을 작성할 필요는 없다. 아래처럼 짧게 요청해도 에이전트는 목표/수정 범위/완료 조건/검증으로 확장해서 처리한다.

운영 중 실수가 확정되면 에이전트는 `docs/HarnessMistakes/README.md`를 사용해서 domain/category를 분류하고 해당 파일을 갱신한다.

## 기본 템플릿

```text
작업 목표:
- [원하는 기능/버그 수정 한 줄]

배경:
- [현재 증상 또는 목적]
- [관련 시스템/파일 경로]

수정 허용 범위:
- [예: Assets/01_Scripts/]
- [필요 시 Assets/99_Tests/]

수정 금지 범위:
- [예: 씬/프리팹/ScriptableObject/ProjectSettings]

완료 조건:
- [조건 1]
- [조건 2]

순서:
- 관련 파일 전부 읽고 분석한 다음 수정해.

검증:
- AGENTS.md 기준으로 검증해줘.
- Unity 에디터가 닫혀 있으면 batch 검증을 실행해줘.
- Unity 에디터가 열려 있으면 내가 실행할 메뉴와 기대 결과를 같이 알려줘.

보고 방식:
- 변경 파일
- 검증 결과
- 남은 리스크나 수동 확인 항목
```

## 템플릿 1: 코드 전용 작업

런타임 코드와 테스트 안에서만 작업할 때.

```text
[기능/버그]를 수정해줘. 관련 파일 전부 읽고 분석한 다음 수정해.
AGENTS.md 규칙을 따르고, 수정은 Assets/01_Scripts/ 안에서만 해줘.
완료 조건은 [조건]이고, 검증은 가능한 자동으로 수행해줘.
Unity 에디터가 열려 있으면 내가 실행할 메뉴와 기대 결과를 같이 알려줘.
```

## 템플릿 2: 코드 + 데이터 작업

ScriptableObject 또는 config asset 편집이 포함될 때. 명시적 허가 필요.

```text
작업 목표: [한 줄]
ScriptableObject 편집이 필요합니다: Assets/12.ScriptableObject/[경로]
수정 허가: 위 경로의 asset 파일 편집을 허용합니다.
```

## 템플릿 3: 씬 또는 프리팹 작업

guarded serialized asset을 의도적으로 허용할 때만 사용.

```text
작업 목표: [한 줄]
씬/프리팹 편집이 필요합니다: [경로]
수정 허가: 위 경로의 편집을 허용합니다.
이유: [왜 코드로 해결할 수 없는지]
```

## 템플릿 4: 조사 우선 (역방향 프롬프팅)

분석만 원하고 코드 변경은 아직 원하지 않을 때.

```text
작업 목표: [알고 싶은 것 또는 풀고 싶은 문제 한 줄]

역할:
- 아직 코드를 수정하지 마.
- 작업을 시작하기 전에 필요한 컨텍스트를 파악할 수 있게
  질문을 3~4개 해줘. 한 번에 하나씩 물어봐.
- 질문이 끝나면 파악한 내용을 요약하고 작업 계획을 제안해줘.
- 계획을 내가 승인하면 그 때 실행에 들어가.
```

예: `전투 시스템이 이상하게 동작해. 고치기 전에 질문 3~4개 해줘. 한 번에 하나씩.`

## 템플릿 5: 하네스 업데이트

```text
하네스 킷을 업데이트해줘.
킷 경로: unity-agent-kit
```

## 템플릿 6: Cross-Agent Handoff 확인

```text
Codex 작업 내역 확인해줘.
docs/AgentHandoffs/pending/codex-to-claude/ 만 먼저 확인하고,
unread handoff note가 있으면 관련된 것만 읽어줘.
이해한 note는 docs/AgentHandoffs/consumed/codex-to-claude/ 로 이동해줘.
없으면 handoff note가 없다고만 알려주고 다른 handoff 파일은 읽지 마.
```

## 템플릿 7: Delegation 허용

```text
이 작업은 delegation을 허용할게.
메인 에이전트가 critical path와 최종 검증을 직접 맡고,
sub-agent는 read-only 조사, 테스트/로그 확인, 명확히 분리된 코드 범위에만 써줘.
같은 파일을 여러 에이전트가 동시에 수정하지 말고, guarded asset은 sub-agent에게 맡기지 마.
```
